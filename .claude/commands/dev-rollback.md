# ロールバック処理

引数で受け取ったBeadsタスクID: $ARGUMENTS に対して、以下のロールバック処理を実行する。

ロールバックとは、タスクの再試行を構造的に管理する仕組みである。

## ステップ1: 旧タスクのクローズ
Agent(`beads-manager`): 旧タスクをクローズする（失敗記録付き）

## ステップ2: 新タスクの作成
Agent(`beads-manager`): 同一要件の新タスクを作成する
- 旧タスクへのリンクを `discovered-from` 依存関係で設定
- 旧タスクで失敗した内容を新タスクの説明に記載
- 「旧タスクで試した方法以外で実装すること」を明記

## ステップ3: 依存関係の付け替え
Agent(`beads-manager`): 旧タスクの前後の依存関係を新タスクへ付け替える

## ステップ4: featureブランチの破棄
Agent(`git-manager`): featureブランチの変更を破棄する

## ステップ5: ロールバック回数の確認
- 3回目まで → 「ロールバック完了、新タスクで再開可能」と報告
- 4回目以降 → **処理を停止し、ユーザーに通知する**
  - 「このタスクは3回ロールバックされました。自動処理の限界です。手動での対応をお願いします。」
