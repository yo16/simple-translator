# 設計フェーズ

以下の手順を順番に実行してください。

## ステップ1: 技術スタック確認
CLAUDE.md の「技術スタック」セクションを読み、フレームワーク（Next.js App Router）と構成（Next.js + ローカル Node.js WebSocket サーバー、GCP 3サービス）を確認する。

## ステップ2: フレームワークスペシャリストに相談
Agent(`nextjs-specialist`) を呼び出し、アーキテクチャ方針の助言を得る。

相談内容: ディレクトリ構造、Next.js(App Router) とローカル Node.js WebSocket サーバーの分離方針、クライアント/サーバー間のデータフロー、音声チャンクの送受信パターン

## ステップ3: 設計ドキュメント作成
Agent(`design-architect`) を呼び出し、要件定義とスペシャリストの助言を踏まえて設計ドキュメントを作成させる。
- 要件定義の配置先は CLAUDE.md の「要件定義ドキュメント」セクションを参照
- 作成するドキュメント一覧は CLAUDE.md の「設計ドキュメント構成」セクションを参照

## ステップ4: ユーザーに設計レビュー依頼
設計ドキュメントの概要をユーザーに提示し、レビューと承認を求める。
不明点や変更要望があれば、設計エージェントに修正を依頼する。

## ステップ5: タスク分解
ユーザーの承認後、Agent(`beads-manager`) を呼び出し、以下を実行させる:
1. 設計ドキュメントからタスクを分解
2. Epic → 子タスクの階層構造を作成
3. タスク間の依存関係を設定（共通型/メッセージ仕様 → WebSocketサーバー → フロントエンドが基本）
4. `bd dep tree` で依存関係を確認
5. `bd ready` で初期実行可能タスクを報告

設計フェーズ完了後、`/dev-start` で開発フェーズに進む。
