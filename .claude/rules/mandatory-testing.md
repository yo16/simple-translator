# テスト必須ルール（絶対厳守）

## 原則: テストなしのタスク完了は認めない

コードを実装したタスクは、**必ず**テスト実装→テストレビュー→テスト実行→テスト結果判定のすべてを通過しなければならない。

## 禁止事項

以下の行為は**いかなる理由があっても禁止**する:

1. **テストステップの省略**: 「簡単な変更だから」「UIだけだから」「設定ファイルだけだから」等の理由でテストを省略すること
2. **手動確認への委任**: 「`npm run dev`で確認してください」「ブラウザで動作確認してください」等、ユーザーに手動テストを委ねること
3. **テスト実装の先送り**: 「後でテストを追加します」「次のタスクでテストします」等の先送り
4. **テスト失敗の無視**: テストが失敗した状態でタスクを完了とすること

## テスト手段の選択

タスクの内容に応じて、以下のテスト手段を**組み合わせて**使用する:

| テスト対象 | 手段 | 例 |
|---|---|---|
| ビジネスロジック | 単体テスト (Jest) | 関数の入出力、バリデーション |
| APIエンドポイント | 結合テスト (Jest + supertest) | レスポンスコード、レスポンス形式 |
| Reactコンポーネント | コンポーネントテスト (React Testing Library) | レンダリング、ユーザー操作 |
| ページ遷移・フォーム | E2Eテスト (Playwright) | ユーザーフロー、画面遷移 |
| CSSスタイリング | スナップショットテスト + 目視用スクリーンショット | レイアウト崩れ検出 |
| DB操作 | 結合テスト (Jest + テストDB) | CRUD操作、RLSポリシー |
| 設定ファイル変更 | バリデーションテスト | 設定値の型・範囲チェック |

## PMロール（メイン会話）への指示

`/dev-task`パイプラインを実行する際、あなた（PM）は以下を守ること:

1. **ステップ4〜7は必ず実行する**: 実装(ステップ2)→コードレビュー(ステップ3)の後、テスト実装(ステップ4)を**必ず**開始すること
2. **テストエージェントを必ず呼び出す**: `frontend-test-engineer`または`backend-test-engineer`のAgent呼び出しを省略しないこと
3. **テスト結果を必ず判定させる**: `frontend-test-judge`または`backend-test-judge`のAgent呼び出しを省略しないこと
4. **テストOKなしにステップ8に進まない**: テストジャッジがOKを出すまで、タスク完了(ステップ8)に進んではならない

## 違反時の対応

このルールに違反してタスクを完了した場合、そのタスクの完了は無効とする。テストを実施し直すこと。
