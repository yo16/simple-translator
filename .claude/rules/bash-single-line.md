# Bashコマンド実行ルール（厳守）

## ルール1: コマンドチェイン禁止
`&&`, `;`, `|` 等で複数のコマンドをチェインして実行してはならない。1つずつ個別に実行すること。

## ルール2: 単一行実行
Bashコマンドは必ず単一行で実行すること。ヒアドキュメント（`<<EOF`）、バッククォート内改行、`$(...)`内改行はすべて禁止。

## ルール3: 複数行引数は外部ファイル経由
複数行のテキストが必要な場合は `tmp/` に一時ファイルを書き出し、コマンドからファイルを参照する。

- `git commit -F tmp/commit-msg.txt`（`-m` は使わない）
- `bd create --body-file tmp/bd-body.md`（`-d` は使わない）
- `bd update {id} --body-file tmp/bd-body.md`
